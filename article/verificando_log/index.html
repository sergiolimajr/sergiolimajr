Formas de verificar o log no Git

No dia a dia precisamos estar atentos ao commits feitos por nós mesmos e também pelos nossos colegas de trabalho. Isso ajuda muito no controle do que está sendo feito no projeto, então decidi fazer esse breve artigo sobre algumas formas legais de verificar o log no Git através do terminal utilizando $ git log.

Utilizando git log

Com o $ git log, podemos ver todos os commits feitos, sendo vistos em ordem descrescente, ou seja, do mais novo commit ao mais antigo.

Inicialmente, usando o comando $ git log -p, vemos cada parte do commit, o que foi mudado, as diferenças entre um arquivo e outro. Basicamente esse comando mostra todos os commits e as alterações feitas neles.

Se quiséssemos ver algumas interações, poderíamos usar $ git log -p -2, trazendo assim os dois ultimos commits, ou quantos você desejar.


commit c192eb18ed3de7888de93bdde938e0bac1c7d55d
Author: jorgebuxe <email@email.com>
Date:   Mon Mar 27 00:03:30 2017 -0300

    Alterando o arquivo.txt

diff --git a/arquivo.txt b/arquivo.txt
index 980a0d5..f534deb 100644
--- a/arquivo.txt
+++ b/arquivo.txt
@@ -1 +1 @@
-Hello World!
+Hello World.

commit 6b9bd6ff28443c98e10f49191862811fce47ea8e
Author: jorgebuxe <email@email.com>
Date:   Sun Mar 26 23:57:28 2017 -0300

    Meu primeiro commit

diff --git a/arquivo.txt b/arquivo.txt
new file mode 100644
index 0000000..980a0d5
--- /dev/null
+++ b/arquivo.txt
@@ -0,0 +1 @@
+Hello World!

Outra coisa que você também pode fazer é usar o $ git log - -stat. Esse comando mostra um resumo estatístico de todos os commits, dizendo quantas linhas foram alterados, quantas foram retiradas e quantas foram adicionadas, simples não é?


commit c192eb18ed3de7888de93bdde938e0bac1c7d55d
Author: jorgebuxe <email@email.com>
Date:   Mon Mar 27 00:03:30 2017 -0300

    Alterando o arquivo.txt

 arquivo.txt | 2 +-
 1 file changed, 1 insertion(+), 1 deletion(-)

Já quando não queremos ver todo esse emaranhado de código, usamos o comando $ git log - -pretty=oneline. Ele retornará tudo numa linha, porém, retirando alguns detalhes.

c192eb18ed3de7888de93bdde938e0bac1c7d55d Alterando o arquivo.txt
6b9bd6ff28443c98e10f49191862811fce47ea8e Meu primeiro commit

Também existe uma outra opção do (- -pretty), que é o (- -pretty=format) seguido do formato que queremos. Usando o comando $ git log - -pretty=format:“%h - %an, %ar : %s” ele traz os commits formatados como desejamos deixando tudo bem organizado pelo hash, quem commitou, há quanto tempo commitou, e qual é a descrição do commit.

   %h         %an          %ar                    %s
-------   ---------  ----------------   -----------------------
c192eb1 - jorgebuxe, 84 minutos atrás : Alterando o arquivo.txt
6b9bd6f - jorgebuxe, 2 horas atrás : Meu primeiro commit

Podemos ainda pegar o log dos commits feitos há dias atras, tanto como minutos, horas. Usando comando $ git log - -sice=2.days ou $ git log - -since=120.minutes que equivale a 2 horas.

commit c192eb18ed3de7888de93bdde938e0bac1c7d55d
Author: jorgebuxe <email@email.com>
Date:   Mon Mar 27 00:03:30 2017 -0300

    Alterando o arquivo.txt

commit 6b9bd6ff28443c98e10f49191862811fce47ea8e
Author: jorgebuxe <email@email.com>
Date:   Sun Mar 26 23:57:28 2017 -0300

    Meu primeiro commit

Conclusão

Então, essas foram algumas dicas de como verificar o log no git. Se não sabia, agora sabe. Compartilhe com seu colega, pratique, vá fundo no git pois não é difícil, e ajuda bastante no versionamento.
